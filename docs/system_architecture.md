## 模块划分

### 1. 设备层 (Devices)

**职责**: PLC 设备连接、数据采集、设备管理

**文件结构**:
```
src/devices/
├── device_config.py      # 设备配置定义
├── device_manager.py     # 设备管理器（核心）
├── connection_manager.py # 连接池管理
└── __init__.py           # 模块导出
```

**核心类**:
- `DeviceManager` - 设备注册、连接管理、数据采集调度
- `PLCClient` - 单个 PLC 设备客户端
- `DeviceCollector` - 数据采集器
- `ConnectionPool` - 连接池（统一管理所有连接）
- `ConnectionStats` - 连接统计

### 2. 数据层 (Data)

**职责**: 数据存储、缓存、持久化

**文件结构**:
```
src/data/
└── data_storage.py       # SQLite数据库操作
```

**核心特性**:
- 异步写入（后台线程批量写入）
- LRU 缓存（减少数据库查询）
- 数据清理（自动删除旧数据）
- 数据库优化（VACUUM/ANALYZE）

### 3. 分析层 (Analysis)

**职责**: 故障检测、异常分析、规则引擎

**文件结构**:
```
src/analysis/
├── fault_detector_base.py      # 故障检测器基类
├── configurable_fault_detector.py # 可配置检测器
├── rxb800_fault_detector.py    # RXB800专用检测器
├── rxa1300_fault_detector.py   # RXA1300专用检测器
├── fault_tracker.py            # 故障跟踪器
├── data_analyzer.py            # 数据分析器
├── drools_lite_engine.py       # 轻量级规则引擎
├── slider_down_detector.py     # 滑块检测
└── rxb800_rules.py            # RXB800规则定义
```

### 4. 服务层 (Services)

**职责**: 业务逻辑协调、数据处理管道

**文件结构**:
```
src/services/
└── data_processor.py      # 数据处理器（业务核心）
```

**核心功能**:
- 数据采集调度
- 故障检测协调
- 异常分析触发
- 数据持久化管理

### 5. API层 (API)

**职责**: REST API 和 SocketIO 实时通信

**文件结构**:
```
src/api/
└── routes.py              # API路由定义
```

**API端点**:

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/devices` | GET | 获取设备列表 |
| `/api/devices/<id>` | GET | 获取设备详情 |
| `/api/data` | GET | 查询历史数据 |
| `/api/anomalies` | GET | 查询异常记录 |
| `/api/faults` | GET | 获取活跃故障 |
| `/api/errors` | GET | 获取错误日志 |
| `/api/storage/stats` | GET | 存储统计 |

### 6. 实时通信层 (SocketIO)

**职责**: 实时数据推送、客户端订阅管理

**文件结构**:
```
src/socketio_handler/
├── events.py                 # SocketIO事件定义
└── subscription_manager.py   # 订阅管理器
```

**事件列表**:

| 事件 | 方向 | 说明 |
|------|------|------|
| `connect` | 客户端→服务端 | 客户端连接 |
| `disconnect` | 客户端→服务端 | 客户端断开 |
| `subscribe` | 客户端→服务端 | 订阅设备/标签 |
| `unsubscribe` | 客户端→服务端 | 取消订阅 |
| `request_full_snapshot` | 客户端→服务端 | 请求完整数据快照 |
| `data` | 服务端→客户端 | 推送增量数据 |
| `fault_update` | 服务端→客户端 | 故障状态更新 |
| `ping` | 双向 | 心跳检测 |

### 7. 启动管理 (Startup)

**职责**: 系统启动流程、状态管理、优雅关闭

**文件结构**:
```
src/startup/
└── startup_manager.py      # 启动管理器
```

**启动阶段**:
1. 初始化数据库
2. 创建设备管理器
3. 初始化设备配置
4. 创建 Flask 应用
5. 注册路由
6. 启动后台服务
7. 启动后台连接
8. 进入运行状态

### 8. 工具层 (Utils)

**职责**: 通用工具、异常处理、验证

**文件结构**:
```
src/utils/
├── error_handling.py       # 全局异常处理
└── validation.py           # 配置验证和运行时检查
```

**核心功能**:
- `GlobalErrorHandler` - 全局异常捕获和记录
- `safe_execute()` - 安全执行函数
- `retry_with_backoff()` - 带退避策略的重试
- `ConfigValidator` - 配置验证器
- `RuntimeChecker` - 运行时检查器

### 9. 序列化层 (Serialization)

**职责**: 数据序列化、压缩、增量传输

**文件结构**:
```
src/serialization/
└── data_serializer.py      # 数据序列化器
```

**核心功能**:
- 支持 JSON 和 MessagePack 序列化
- zlib 压缩
- 增量数据计算
- 统一数据包格式

## 数据流

### 数据采集流程

```
PLC设备 → PLCClient → DeviceCollector → DataProcessor → DataStorage
                                              ↓
                                      SocketIO推送 → 客户端
```

### 故障检测流程

```
DataProcessor → FaultDetectorRegistry → 检测规则匹配 → FaultTracker → 数据库记录
                                                              ↓
                                                      SocketIO推送故障更新
```

## 配置系统

### 环境变量支持

所有配置项都可以通过环境变量覆盖：

```bash
PLC_HOST=192.168.1.100 SERVER_PORT=8080 python run.py
```

### 配置文件

支持 `.env` 文件配置：

```env
PLC_HOST=172.15.14.150
SERVER_PORT=3000
DATA_HISTORY_DAYS=30
```

### 配置分组

| 分组 | 说明 |
|------|------|
| PLC | PLC连接配置 |
| DATA | 数据采集配置 |
| ANALYSIS | 分析参数配置 |
| SERVER | 服务端配置 |
| LOGGING | 日志配置 |
| DATABASE | 数据库配置 |
| CONNECTION_POOL | 连接池配置 |
| HEALTH_CHECK | 健康检查配置 |
