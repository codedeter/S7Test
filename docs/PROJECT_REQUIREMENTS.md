# 多生产线PLC监测系统 - 项目需求文档

## 一、项目概述

### 1.1 项目背景

本项目是一个**多生产线PLC（可编程逻辑控制器）实时监测系统**，用于监控工厂多条生产线上的PLC设备状态和数据。

### 1.2 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端展示层                            │
│                    (Web监控界面)                             │
│              http://localhost:3000                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       服务器层                               │
│              (Flask + SocketIO + Data Pipeline)            │
│                  端口: 3000                                 │
│          职责: 数据聚合、WebSocket推送、API服务              │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              │               │               │
              ▼               ▼               ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│    生产线A      │ │    生产线B      │ │    生产线C      │
│  子网: 172.15.x.x │ │  子网: 172.16.x.x │ │  子网: 172.17.x.x │
│  - PLC_001     │ │  - PLC_002     │ │  - PLC_003     │
│  - PLC_002     │ │  - PLC_004     │ │  - PLC_005     │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 2.2 连接可靠性需求

#### 2.2.1 设备状态处理

系统必须能够优雅处理以下情况：

| 场景 | 预期行为 | 优先级 |
|------|---------|--------|
| 设备正常在线 | 实时采集和显示数据 | P0 |
| 设备关机 | 显示离线状态，不影响其他设备 | P0 |
| 设备断电 | 显示离线状态，停止数据采集 | P0 |
| 设备损坏 | 显示故障状态，标记需要维护 | P0 |
| 网络故障 | 显示网络错误，尝试重连 | P0 |
| 部分子网不可达 | 只显示可达设备，其他显示离线 | P0 |

#### 2.2.2 重连策略

```
连接失败后的重连间隔策略：
├── 第1次失败：立即重试
├── 第2次失败：等待 5秒
├── 第3次失败：等待 10秒
├── 第4次失败：等待 20秒
├── 第5次失败：等待 40秒
├── 第6次+失败：挂起该设备，5分钟后重试
└── 挂起期间：标记设备为"已挂起"，减少资源消耗
```

#### 2.2.3 健康状态监控

| 指标 | 说明 | 阈值 |
|------|------|------|
| 连接成功率 | 成功连接次数/总尝试次数 | > 95% |
| 平均响应时间 | PLC响应时间 | < 100ms |
| 数据采集完整性 | 成功采集次数/应采集次数 | > 99% |
| 设备在线率 | 在线设备数/配置设备数 | 动态显示 |

### 2.3 数据采集需求

#### 2.3.1 采集参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 采集间隔 | 100ms | 数据采样周期 |
| 连接超时 | 15秒 | PLC连接超时时间 |
| 读取超时 | 5秒 | 数据读取超时时间 |
| 最大并发 | 10个设备 | 同时采集的设备数 |

#### 2.3.2 数据类型

| 数据类型 | 说明 | 示例 |
|---------|------|------|
| Bool变量 | 位信号状态 | 开关状态、报警信号 |
| Int变量 | 16位整数 | 计数器、模式选择 |
| DInt变量 | 32位整数 | 时间值、大数值 |
| Real变量 | 浮点数 | 压力、温度、速度 |

#### 2.3.3 数据块定义

每个PLC设备包含以下数据块：

| DB号 | 数据类型 | 说明 |
|------|---------|------|
| DB1 | Bool/Int | 控制信号、状态位 |
| DB10 | Real/Int | 实时数据（压力、速度等） |
| DB51 | Bool | 故障报警状态 |

## 三、技术要求

### 3.1 性能指标

| 指标 | 要求 | 实际 |
|------|------|------|
| 启动时间 | < 5秒 | ~2.5秒 |
| 页面响应时间 | < 1秒 | < 0.5秒 |
| 数据更新延迟 | < 200ms | ~100ms |
| 最大设备数 | 50+ | 支持 |
| 数据存储 | > 7天 | SQLite |

### 3.2 稳定性要求

| 场景 | 要求 |
|------|------|
| 24小时运行 | 无内存泄漏 |
| 设备频繁开关机 | 自动重连，不崩溃 |
| 网络波动 | 自动恢复 |
| 异常数据 | 过滤和日志记录 |

### 3.3 错误处理

```python
# 统一的错误处理策略
错误分类：
├── 网络错误 (NetworkError)
│   ├── 超时
│   ├── 连接被拒绝
│   └── 主机不可达
├── 协议错误 (ProtocolError)
│   ├── 数据校验失败
│   └── 无效响应
├── 设备错误 (DeviceError)
│   ├── 设备无响应
│   ├── 数据块不存在
│   └── 读取失败
└── 系统错误 (SystemError)
    ├── 资源不足
    └── 线程异常
```

### 5.2 配置说明

**重要：IP地址为设备常量，由现场工程师配置，不可随意修改**

系统需要能够：
1. **识别不可达设备** - 自动检测并标记
2. **优雅降级** - 单个设备故障不影响整体
3. **智能重连** - 在设备恢复后自动重连
4. **状态显示** - 清晰展示每个设备的状态

### 6.3 常见问题处理

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 设备一直离线 | IP不可达/设备关机 | 确认设备电源和网络 |

| 连接不稳定 | 网络波动 | 检查网络配置 |
